Project Overview

PondPedia is an advanced Android application built using Kotlin and Jetpack Compose, designed with clean architecture principles and modern best practices. The app provides a user-friendly interface for comprehensive pond and aquatic ecosystem management and information.

Key Features

  • Management: Add, edit, and delete pond profiles with detailed information such as location, size, and species.
  • Water Quality Monitoring: Record and track key water parameters like temperature, pH, and dissolved oxygen.
  • Feeding Management: Create and manage feeding schedules with customizable feeding amounts and frequencies.
  • Growth Tracking: Monitor the growth of aquatic life by recording weight and length measurements.
  • AI Chat: Consult with an AI-powered chatbot for expert advice on pond management.
  • Intuitive User Interface: Crafted using Jetpack Compose for seamless UI development and enhanced user experience.
  • Authentication & Security: Secure user authentication implemented via Firebase Authentication, supported by custom Auth Interceptors.
  • Local Data Management: Robust local data handling using DataStore and Room database for offline support.
  • RESTful API Integration: Managed through Retrofit with custom interceptors and authenticators for secure API requests.

Technology Stacks

  • Programming Language: Kotlin
  • Framework: Android Jetpack Compose
  • Libraries:
    • Dagger-Hilt for dependency injection
    • Room for local database management
    • Retrofit for network communication
  • Tools:
    • Android Studio for development
    • GitHub for version control

Architecture and Design Patterns

Pondpedia follows the MVVM (Model-View-ViewModel) and Clean Architecture to ensure a clear separation of concerns and improve code testability. Key design patterns used in the project include:

  • Repository Pattern: Provides an abstraction layer for data access, allowing for seamless switching between local and remote data sources.
  • Singleton Pattern: Ensures that only one instance of certain classes, such as the database instance, exists throughout the application.

Challenge and Solutions

Challenge: Efficiently managing and displaying large datasets of pond records. Solutions: Implemented Room database with optimized queries and data structures to ensure smooth performance.

Challenge: Integrating the Authorization within the application. Solutions: Custom AuthInterceptor and TokenManager were implemented to maintain secure and consistent authentication states.

Challenge: Integrating the AI chatbot with the application's UI. Solutions: Utilized Jetpack Compose's state management capabilities to seamlessly update the UI with chatbot responses.
